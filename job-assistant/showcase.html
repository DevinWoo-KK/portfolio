Why

求职不应该靠刷招聘 App

重复手动搜岗位、逐条判断是否匹配、手动编辑打招呼语、逐个点击投递
—— 这个流程 AI 可以做得更好、更快、不会漏。

😤

痛点

每天打开三四个招聘 App，重复搜关键词，手动判断岗位是否匹配，编辑个性化话术。3 个平台 × 每天 2 次 = 巨大的时间黑洞。而且容易漏掉好岗位。

🤖

方案

定时扫描三平台热搜关键词 → 按简历自动匹配打分 → 自动生成个性化打招呼话术 → 自动投递 → 数据追踪。你只需要看日报和准备面试。

Cookie 注入 · 免登录自动化
三平台统一方案：从 Firefox cookies.sqlite 读取登录态 Cookie，Playwright 注入后直接操作搜索和投递页面，完全绕过验证码。Cookie 过期时 Telegram 自动告警，手动扫码后 30 秒恢复运行。
PlaywrightFirefoxSQLiteTelegram告警
Boss PUA 薪资解码
Boss 直聘使用 Private Use Area (U+E031-U+E03A) 编码薪资数字，防止机器抓取。通过 JavaScript 范围内解码 decoded = char - 0xE031 + 0x30，100% 还原真实薪资。比字典映射方案更健壮，不会丢字符。
Unicode PUAJS逆向薪资解码
错峰反爬 · 账号权重保护
扫描、打招呼、监控三个 cron 任务分时段执行，每次间隔 5-10 秒。Boss 打招呼 单账号每日上限 20 个，超出自动暂停次日继续。猎聘分时段执行避免 IP 风控。三平台登录态独立维护，互不影响。
Cron分时速率限制权重保护
Profile 自治架构
求职助手作为独立 Hermes Agent Profile 运行，完全隔离：独立的 SOUL.md、MEMORY.md、配置文件和 session 数据库。12 个 cron 任务通过 workdir 指向 profile 目录自动注入上下文，主 agent 零负担。
